2. GPIO 腳位配置

輸入腳位

34
藍牙偵測 HIGH = 已連線
35
ACC / 點火偵測 HIGH = 引擎啟動

輸出腳位

5
鎖門 Relay Active LOW
15
解鎖 Relay Active LOW
4
發車 Relay Active LOW
26
鑰匙電源 (POWER) HIGH = 通電
27
備用電源 (R1) BT 連線時通電

7. 動作執行時序

triggerBoot() - 引擎啟動

Guard
if ACC==HIGH → return
Flag
is_acting = true
+0ms
POWER_PIN = HIGH
+100ms
RELAY_LOCK = LOW (觸發)
+500ms
RELAY_LOCK = HIGH (釋放)
+800ms
RELAY_BOOT = LOW (啟動馬達)
+3800ms
RELAY_BOOT = HIGH, POWER = LOW
Done
is_acting = false

unlockDoor() - 解鎖

Guard
if ACC==HIGH → return
Flag
is_acting=true, preAct=1
+0ms
POWER_PIN = HIGH
+100ms
RELAY_OPEN = LOW
+300ms
RELAY_OPEN = HIGH
+400ms
POWER_PIN = LOW
HTTP
SendCarPowerMsg(3)

lockDoor() - 鎖門

Guard
if ACC==HIGH → return
Flag
is_acting=true, preAct=0
+0ms
POWER_PIN = HIGH
+100ms
RELAY_LOCK = LOW
+300ms
RELAY_LOCK = HIGH
+400ms
POWER_PIN = LOW
HTTP
SendCarPowerMsg(2)

closeWindow() / OpenWindow() - 車窗控制

Guard
if ACC==HIGH → return
第一段
POWER ON → Relay 脈衝 200ms (觸發短按)
間隔

第二段
Relay 長按 (關窗: close_window_delay / 開窗: open_window_delay)
HTTP

8. 狀態機分析

核心狀態變數

變數類型初始值意義
preActint0門是否已解鎖 (1=已解鎖, 0=未解鎖/已鎖)
autoUnlockTriggeredboolfalse本次 BT 連線週期是否已自動解鎖
is_actingboolfalse繼電器動作進行中 (鎖定 POWER_PIN 控制)
key_linkint0MQTT key 連動模式 (1=啟用自動解鎖)
lastBlueStateint-1藍牙前一狀態 (debounce 用)
lastAccStateint讀取值ACC 前一狀態 (debounce 用)

自動解鎖/鎖門 狀態轉移圖

stateDiagram-v2
    [*] --> 待機: 開機

    待機 --> 已解鎖: BT=HIGH + key_link=1\n+ ACC=LOW + preAct=0\n→ unlockDoor()

    已解鎖 --> 已解鎖: BT 維持 HIGH\nPOWER_PIN 持續供電

    已解鎖 --> 待機: BT HIGH→LOW 斷線\n→ lockDoor() + 重置

    已解鎖 --> 待機: MQTT key_off\n→ 重置 flag + POWER LOW

    待機 --> 待機: BT=LOW 或 key_link=0\n保持待機

    note right of 已解鎖
        autoUnlockTriggered = true
        preAct = 1
        POWER_PIN = HIGH
    end note

    note right of 待機
        autoUnlockTriggered = false
        preAct = 0
        POWER_PIN = LOW
    end note

完整狀態轉移表

當前狀態 BT ACC key_link preAct autoUnlock 觸發事件 動作 次態
待機 LOWanyanyanyfalse —— 待機
待機 HIGHLOW10false 條件齊備 unlockDoor() 已解鎖
已解鎖 HIGHLOW11true 維持連線 POWER_PIN HIGH 已解鎖
已解鎖 HIGH→LOW——1true BT 斷線確認 lockDoor() + 重置 待機
已解鎖 ——01true MQTT key_off preAct=0, flag=false, POWER LOW 待機
任意 anyHIGHanyanyany ACC ON 所有動作函式 return 不動作

安全聯鎖保護機制

機制保護目的實作方式
ACC == HIGH → return 引擎運行中禁止所有繼電器操作 所有動作函式開頭檢查
is_acting flag 防止動作執行中被 GPIO 邏輯干擾 POWER_PIN 控制區塊跳過
autoUnlockTriggered 每次 BT 連線只自動解鎖一次 解鎖後設 true, BT 斷線時重置
50ms debounce 避免 GPIO 雜訊造成誤判 BT / ACC 邊緣偵測前 delay + 二次確認

10. OBD 資料轉發流程

sequenceDiagram
    participant Device as OBD 裝置
    participant WS as WebSocket:81
    participant ESP as ESP32
    participant MQTT as OBD MQTT Broker

    Device->>WS: 連線 (WStype_CONNECTED)
    Note over WS,ESP: 記錄來源 IP

    loop 持續傳送
        Device->>WS: 傳送 OBD 數據 (WStype_TEXT)
        WS->>ESP: 觸發 webSocketEvent
        alt OBD MQTT 已連線
            ESP->>MQTT: publish(OBD_MQTT_TOPIC, data)
            Note over MQTT: 數據已轉發
        else OBD MQTT 未連線
            Note over ESP: 丟棄，Serial 印出警告
        end
    end

    Device->>WS: 斷線 (WStype_DISCONNECTED)
